Make Your Own URL Shortener With Rails 4 and Heroku

Feb 26th, 2014

Ever wanted to make your very own URL shortener? Know a little bit of rails? Then you’re good to go!

Here is what I ended up making: http://acal.io/”>acal.io

Set Up

Make a new rails app with pg since I’m deploying to heroku

add pry to gemfile

Download bootstrap and unzip it.

Add bootstrap.min.js to the folder vendor/assets/jacascripts. Then modify application.js to indicate this

Add bootstrap.min.css to the folder vendor/assets/stylesheets. Then modify application.css to indicate this

Models

Generate the scaffolding for the link

Modify the default value for clicks BEFORE migrating. This is because you can’t increment a non-integer value (nil), which is the original default.

Make a method to generate the slug aka the short part of the link after the base url. I’m using base 36, which allows really large numbers to be shown in just a few characters. For example the 1 Billionth link in the database will have a slug of “gjdgxs” - just 6 characters as opposed to the 10 characters in the number 1,000,000,000.

Add the figaro gem

Then generate the application.yml file that git will ignore (so your passwords don’t get committed to the rest of the world)

Then add the base url as an environmental variable to the application.yml file

Make a convenience method to display the entire shortened url

I know I want to capture a screenshot as well as the page title. I will use IMGKit, CarrierWave and Fog for the screenshot and Mechanize for getting the page title. I also know upfront that I want to do this asynchronously, so I’ll also throw in SideKiq.

Make Amazon Environmental Variables

Configure CarrierWave

Now I can actually implement the screenshot in the link class

First I add the upload functionality of CarrierWave

Then I make a method for actually getting the screenshot and page title. For now, these are just Sidekiq Workers which I’ll actually build later.

I then make sure to create the slug, capture the screenshot, and capture the page title by making after_create callbacks.

I then make my screenshot worker

Then I make my scrape worker

I finish up by making the uploader that CarrierWave needs. I am adding the cache_dir specifically to get around Heroku’s file writing permissions issues.

Controllers

First up I make some modifications to the auto generated links controller. I decided to delete index, edit, update, and destroy functionality. These get into permissions issues, which could lead naturally to building out users and all sorts of other features that are not core to this app. I also wanted the create action to respond asynchronously, so added the ability to respond to format.js (more on this later).

I also added the ability to route a slug on a naked url to the show action.

This is so the URL http://acal.io/slug will naturally get routed to the long URL that the corresponding slug was made for. It also increments every time the link is clicked. If the user instead goes to http://acal.io/links/id, no incrementing is done, and they are taken to the show page for that link - same as normal.

I also knew that I wanted to render out trending links on root_path of the app, so I generated a home controller.

I modified the routes file to make the index action on the home controller the root

I decided to only have the top 12 trending items, since it’s allows for bootstrap to gracefully degrade from 4 to 3 to 2 to 1 columns. I sliced them into arrays of 4 so that I could make the bootstrap rows down the line using the each_slice method.

I also wanted a page that just listed all the links created as a table, so first made a rotue

I wanted to be sure not to overwhelm the server in case this app received a lot of requests to shorten the url. I opted to use the will_paginate gem

Then I filled out the all action, and decided for now I’ll just put 4 rows per page as the app is small.

Front End

I first made a new partial for a new link that would persist on top. I included a blank div that I wanted to have so I could append the result later on.

I then modified the auto generated title, added the partial site wide and wrapped everything in a bootstrap container

I then modified the link form partial. I wanted to make this submit asyncronously so added a remote:true option to the form. I also added a logo, some placeholder text, and some bootstrap styling.

I then wanted to append the short link result back onto the page once it was processed. I referred to this earlier in the post when I was making the links controller. In order to do this I made a new file

I then made the show page. I added in a bit of formatting, both from bootstrap, and from my own css.

I also wanted to make sure that non-valid URLs could not be submitted. There are a lot of ways to do this, but I settled with doing client side validation.

I first installed the jquery-validation-rails gem

I then added javascript files to application.js

I then made a javascript validation that required both the presence of a url, and it’s validity according to a regex

I then made the site index page

I finally made the all page

Along the way I added a lot of custom css. Of note is an animated yellow flash of the new shortened link.

Deploying to Heroku

I added a few Heroku specific gems

I needed to download a special version of IMGKit.

https://wkhtmltopdf.googlecode.com/files/wkhtmltoimage-0.10.0_rc2-static-amd64.tar.bz2

I unzipped it and put it in the bin directory.

I had to make a configure file for IMGKit so that this would actually take effect.

I also had to modify config.ru to get CarrierWave to work

I then made a Procfile

I then had to configure unicorn

I created, pushed, and migrated the app on Heroku

I then configured the environmental variables

Then I added redistogo and added another dyno so that sidekiq would run

Finally I bought a custom domain through hover. I had to do first add a CNAME record called ‘www’ with the target being my heroku domain. CNAME www example.herokuapp.com.

I also set up domain forwarding so that a naked url would just redirect to www. Heroku does not support naked domains out of the box. There are other solutions out there, but they cost money.

I then associated the domain with my app

Conclusion

That seems like a lot, but it actually doesn’t take that long to do at all. It’s a great learning project for those of you just starting out as a developer. Let me know if you can actually use this to make your own URL shortener, or if there is something I missed. Best of luck!
